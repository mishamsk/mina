# HTTP API Chi and OpenAPI Codegen Migration

## Plan Context

Stage 1 remains REST API only. Keep `api/openapi.yaml` as the REST contract source, `internal/httpapi` as the REST adapter, and service packages as the owners of domain validation and use-case behavior. This plan migrates the HTTP adapter from hand-registered `net/http` routes to Chi routes generated by `oapi-codegen` strict-server mode, adds a Chi middleware stack with access logging, and removes manual HTTP code made redundant by generated routing and binding.
