# Plan: fp3e fix 1 — amounts must never wrap or overlap at narrow transaction-table widths

Fix one live-verified visual regression from the fp3e affordance work: at narrow container widths the transactions-table amount chips wrap onto multiple lines (rows grow 2–3 lines tall) and the mixed-class two-amount chip visibly overlaps adjacent content. Amounts are the most critical data on the line and must stay single-line at every collapse-ladder step.

## Plan Context

- Do not run review-loop.
- Frontend-only. No ground-truth doc edits, no API changes.
- Defect evidence (live, `just dev --demo`, branch as of commit `bc19283`):
  - Viewport 900×800 (container ≈630px, so member+status collapsed, actions folded per the ladder): the income row's `+3,250.00 $` amount chip wraps onto two lines; the mixed-class row's `-5.00 / +100.00 $` renders with glyphs overlapping each other.
