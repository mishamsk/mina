# Plan: d7av fix 1 — page shortcut leaks through the modal palette; listbox ARIA conformance

Fix two operator-review findings on the command palette: the transactions page `n` shortcut can fire while the modal palette is open, and the palette's listbox structure is not ARIA-conformant.

## Plan Context

- Do not run review-loop.
- Frontend-only. No ground-truth doc edits, no API changes.
- Defects (as of commit `8f45152`):
  1. **`n` leaks through the modal palette.** `frontend/src/pages/transactions-page.tsx:206-237` guards inputs/popovers/detail but not the palette. With the palette open on `/transactions`, Tab moves focus onto an option `<button>`; pressing `n` there opens the entry panel behind the modal backdrop. Per `docs/webui-design.md` §Overlays, a modal dialog must make underlying shortcuts inert. Fix: guard the `n` handler on the palette-open store state (readable outside React), or equivalently suppress underlying single-key shortcuts while the palette is open — smallest correct change.
  2. **Listbox structure non-conformant.** `frontend/src/features/command-palette/command-palette.tsx:837-922`: `role="listbox"` children include `<section aria-labelledby>` wrappers (implicit `region` role) and `<h3>` headings; ARIA listbox children must be `option`/`group`. Also `aria-expanded="true"` is hardcoded and `aria-controls="command-palette-results"` dangles in the empty/skeleton branch (`:800-801, 819`). Fix: use `role="group"` + `aria-label` for the sections (visual headings can stay as presentational elements inside the group or become the group labels), and make `aria-expanded`/`aria-controls` reflect reality.
- Protect — do not regress: all palette behavior verified in review (shortcut toggle and gating over popovers/modals, Esc capture without leaking, arrows/Enter with `aria-activedescendant` and scroll-into-view, grouped rendering and visual style unchanged, entity search, entry commands with requested tab + draft-tab preservation, app-action toasts, focus restore semantics); the transactions `n` shortcut when the palette is closed; `just test-frontend-e2e` green (104 tests).
- Scope exclusions: nothing else — no launcher-gating changes, no backup capability probing, no new commands.
